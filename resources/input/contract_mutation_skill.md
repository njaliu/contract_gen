# Skill: 合同 Mutation 要求

## 技能描述

你需要在基准合同（base contract）的基础上进行 mutation 变换，生成不同文本版本。

## 核心约束（必须严格遵守）

以下内容必须严格保持不变，不得删改：

1. 技术指标；
2. 量化技术要求；
3. 评价标准（含验收标准中的量化阈值与判定口径）。

## 允许调整的范围

主要只允许调整以下三部分的文字描述：

1. 研发内容；
2. 研发目标；
3. 产品目标。

## 调整方式要求

在不改变原有事实和约束的前提下，可以进行：

- 同义改写；
- 句式调整；
- 段落组织优化；
- 适当扩展表述（仅做说明性扩展，不新增实质性要求）。

## 禁止事项

- 不得改动任何技术指标、量化技术要求、评价标准；
- 不得新增与原合同冲突的新要求；
- 不得删除关键约束信息。
